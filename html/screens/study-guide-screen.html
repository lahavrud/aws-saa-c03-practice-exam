The AWS Certified Solutions Architect - Associate (SAA-C03) exam validates your ability to design and deploy scalable, highly available, and fault-tolerant systems on AWS. The exam tests your knowledge across four key domains of AWS architecture.

Question Types: Multiple choice and multiple response questions. Some questions may include diagrams or scenarios that require you to select the best solution.

Exam Domains & Weighting

Domain 1: Design Secure Architectures

30%

Key Topics:

  • IAM roles, policies, and best practices
  • Data encryption (at rest and in transit)
  • Network security (VPC, Security Groups, NACLs)
  • AWS WAF, Shield, and GuardDuty
  • Compliance and data protection
  • Secrets management (Secrets Manager, Parameter Store)

Domain 2: Design Resilient Architectures

26%

Key Topics:

  • High availability and fault tolerance
  • Multi-AZ and multi-region deployments
  • Backup and disaster recovery strategies
  • Auto Scaling and Elastic Load Balancing
  • RDS, DynamoDB, and S3 replication
  • Route 53 health checks and failover

Domain 3: Design High-Performing Architectures

24%

Key Topics:

  • Elastic Compute Cloud (EC2) instance types
  • Amazon S3 and CloudFront CDN
  • Database performance optimization
  • ElastiCache and DAX caching strategies
  • Auto Scaling policies and metrics
  • Network performance and placement groups

Domain 4: Design Cost-Optimized Architectures

20%

Key Topics:

  • EC2 pricing models (On-Demand, Reserved, Spot)
  • Savings Plans and Reserved Instances
  • S3 storage classes and lifecycle policies
  • Cost monitoring with Cost Explorer and Budgets
  • Right-sizing instances and resources
  • Data transfer cost optimization

Study Tips & Strategies

1

Start with Hands-On Practice

Create a free AWS account and practice building solutions. Hands-on experience is crucial for understanding AWS services and their interactions.

2

Use Practice Exams

Take multiple practice exams to familiarize yourself with question formats and identify weak areas. Aim for consistent scores above 80% before scheduling your exam.

3

Focus on Weak Domains

Use our domain-specific practice mode to focus on areas where you score lower. Review explanations thoroughly to understand the reasoning behind correct answers.

4

Understand AWS Well-Architected Framework

The five pillars (Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization) are fundamental to the exam.

5

Review AWS Documentation

Read AWS service documentation, especially FAQs and best practices. The exam often tests your knowledge of AWS-recommended approaches.

6

Time Management

Practice completing 65 questions in 130 minutes. That's approximately 2 minutes per question. Use our timed test mode to build this skill.

Essential AWS Services to Master

Compute

  • EC2 (Instance types, AMIs, EBS)
  • Lambda (Serverless functions)
  • Elastic Beanstalk
  • ECS & EKS (Container services)

Storage

  • S3 (Storage classes, lifecycle policies)
  • EBS (Volume types, snapshots)
  • EFS (Elastic File System)
  • Storage Gateway

Database

  • RDS (Multi-AZ, read replicas)
  • DynamoDB (Global tables, DAX)
  • ElastiCache (Redis & Memcached)
  • Redshift

Networking

  • VPC (Subnets, Route Tables, Internet Gateway)
  • CloudFront (CDN)
  • Route 53 (DNS, health checks)
  • API Gateway

Security

  • IAM (Users, Roles, Policies)
  • KMS (Key Management Service)
  • Secrets Manager
  • WAF, Shield, GuardDuty

Management

  • CloudWatch (Monitoring, Alarms)
  • CloudFormation (Infrastructure as Code)
  • Auto Scaling
  • Elastic Load Balancing

Before the Exam

During the Exam

  • Read questions carefully - look for keywords like "MOST", "LEAST", "BEST"
  • Eliminate obviously wrong answers first
  • Mark questions for review if unsure
  • Manage your time - don't spend too long on one question
  • Review marked questions before submitting

Question Strategy

  • Focus on AWS best practices and recommended approaches
  • Consider cost, security, and performance in your answers
